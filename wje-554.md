## 项目主要需求

### 项目概述

构建一个家政与本地生活服务平台（HomeService Hub），连接客户与家政技师，提供在线下单、智能派单、实时通知、服务评价的完整闭环。平台支持三种角色（Customer/Worker/Admin），涵盖清洁、维修、管道、搬家、跑腿五大服务类目。

**项目英文短名（COMPOSE_PROJECT_NAME）**：`wjehomesvc`

**技术栈**：React 18 + TypeScript + Material UI（前端） / NestJS + Prisma（后端） / PostgreSQL / Vite / JWT / WebSocket（实时通知）

**前端端口**：38204，**后端端口**：38304

### 核心实体与字段

| updatedAt | timestamp | 更新时间 |

| 字段 | 类型 | 说明 |

| customerId | UUID | 外键 → User（客户） |
| workerId | UUID | 外键 → Worker（技师），可为空（待派单时） |

| contactPhone | string | 联系电话 |
| scheduledTime | timestamp | 预约上门时间 |
| status | enum | PENDING / ASSIGNED / ACCEPTED / ON_THE_WAY / IN_PROGRESS / COMPLETED / RATED / CANCELLED |
| totalPrice | decimal | 订单总价 |
| actualDuration | int | 实际用时（分钟），完工后填写 |
| rating | int | 评分 1-5，评价后填写 |

| cancelReason | string | 取消原因 |

```
→ Worker完工 → COMPLETED → Customer评价 → RATED

```

**业务动作**：下单、派单、接单、出发通知、开始服务、完工确认、评价、取消。

### 横切关注点

#### 1. 认证授权 + 多角色 RBAC

- **后端**：
  - JWT 认证中间件：验证 Token、解析用户信息
  - Roles Guard：基于角色的路由守卫，装饰器 `@Roles(Role.ADMIN)` 限制访问
  - 数据范围过滤：Customer 只能查看自己的订单，Worker 只能查看分配给自己的订单
- **前端**：
  - 路由守卫：未登录跳转登录页，角色不足跳转 403 页
  - 按钮级权限：根据角色显示/隐藏操作按钮
  - API 请求拦截器：统一附加 JWT Token，401 时自动跳转登录
  - 登录/注册页面：支持角色选择（Customer/Worker 注册）
- **触达层**：DB → Prisma Schema → Service → Controller → Guard → 前端路由 → 按钮可见性 → 数据范围

#### 2. 实时通知（WebSocket）

- **后端 WebSocket Gateway**：
  - 连接认证：通过 JWT Token 验证身份
  - 事件类型：`order:status_changed`、`order:new_assignment`、`order:worker_arriving`
  - 精准推送：根据订单相关方（Customer/Worker）定向发送
- **前端通知组件**：
  - NotificationBell 组件：右上角铃铛图标，未读数量角标
  - 通知下拉面板：展示最近 20 条通知，点击跳转对应订单
  - 浏览器通知：订单状态变更时弹出系统通知（需用户授权）
  - Toast 消息：轻量级状态提示
- **触达层**：后端 WebSocket Gateway → 前端 WebSocket 客户端 → NotificationBell 组件 → 全局状态

#### 3. 全局异常处理

- **后端**：NestJS 全局异常过滤器（ExceptionFilter），统一捕获业务异常和系统异常，返回标准错误响应 `{ code, message, details }`
- **前端**：Axios 响应拦截器，统一处理 4xx/5xx 错误，展示 Toast 错误提示，401 自动跳转登录

#### 4. 操作日志（审计）

- 关键写操作通过审计中间件记录：订单创建、订单状态变更、技师审核、服务上下架、价格修改
- 日志字段：操作人ID、操作类型、目标实体、操作时间、IP 地址、请求参数
- 审计日志表 AuditLog 存储于 PostgreSQL

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=wjehomesvc`，确保 Compose 项目名不依赖目录名
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: wjehomesvc`
- 所有服务使用 `container_name` 命名容器，带 `${COMPOSE_PROJECT_NAME:-wjehomesvc}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（调用 `/api`，由 Nginx 转发到 `backend:38304`）
- 服务间通过 Docker 内部网络通信
- 数据库服务配置 `healthcheck`，后端 `depends_on: { db: { condition: service_healthy } }`
- 后端也配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `38204:80`，后端 `38304:38304`
- 所有可变配置通过环境变量注入
